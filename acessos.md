# Acessos e Primeiros Passos

>  Este guia foi criado para orientar novos colaboradores do suporte sobre como acessar os sistemas, solicitar permissões e iniciar a operação no dia a dia.

---

## Objetivo desta página

Esta seção complementa a Central de Conhecimento com um foco prático em onboarding.

A proposta é reduzir dúvidas recorrentes sobre:

- onde acessar cada sistema
- quem aciona a liberação de acesso
- o que cada sistema entrega no dia a dia
- como validar se o primeiro acesso foi configurado corretamente

> Sempre que algum responsável, canal ou link mudar, esta página deve ser atualizada para manter o onboarding confiável.

---

## Visão geral dos sistemas

| Sistema | Finalidade | Quando é usado |
|---|---|---|
| **SGN** | Operação principal do suporte | Consulta, atendimento e fluxos do dia a dia |

| **Benner** | Consulta de informações financeiras | Conferência de dados financeiros e integrações |

| **Integrações** | Fluxos entre sistemas | Análise de erros técnicos e validações de integração |

---

## Como solicitar acessos

O acesso aos sistemas deve seguir um fluxo simples e padronizado.

1. Identificar quais sistemas serão necessários para a função
2. Confirmar com a liderança ou equipe responsável quais acessos devem ser liberados
3. Solicitar a liberação pelos canais internos definidos pela equipe
4. Aguardar confirmação
5. Validar o acesso recebido antes de iniciar a operação

## O que o colaborador deve entender em cada sistema

### SGN
É o sistema principal de operação. Aqui o colaborador normalmente executa consultas, acompanha rotinas do suporte e valida informações usadas no atendimento.

### Benner
É usado para consulta de informações financeiras. Quando a equipe precisa confirmar dados financeiros ou documentais, este sistema tende a ser a referência.

### Integrações
É a camada que conecta sistemas e fluxos. Geralmente será consultada quando houver erro técnico, falha de sincronização ou necessidade de validar comunicação entre sistemas.

---

## Checklist de primeiro acesso

Antes de considerar o onboarding concluído, validar:

- login funcionando nos sistemas necessários
- permissão compatível com a função
- acesso aos menus principais
- capacidade de consulta no Benner
- conhecimento de onde consultar erros na Central de Conhecimento

---

## Primeiros passos após receber os acessos

1. Validar login em todos os sistemas liberados
2. Navegar pelas telas principais
3. Entender rapidamente a finalidade de cada sistema
4. Consultar esta Central de Conhecimento para conhecer erros recorrentes
5. Acompanhar os primeiros atendimentos com apoio da equipe
6. Registrar qualquer inconsistência de acesso antes de iniciar a operação sozinho

---

## Problemas comuns de acesso

### Usuário sem permissão esperada
O login funciona, mas o colaborador não visualiza menus, filtros ou áreas necessárias.

**Ação recomendada:** validar com o responsável se o perfil liberado está correto.

### Acesso ainda não liberado
O usuário foi criado, mas não consegue entrar no sistema.

**Ação recomendada:** confirmar se a liberação foi concluída e se existe alguma etapa pendente.

### Dúvida sobre onde pedir acesso
Quando não estiver claro qual área ou canal acionar.

**Ação recomendada:** seguir a referência definida pela liderança do time até que a governança esteja formalizada nesta página.

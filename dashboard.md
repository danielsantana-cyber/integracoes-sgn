# Painel de monitoramento

> 📌 O painel não é a frente principal de tratativa. Ele é um ponto de decisão para o suporte.

## 🧭 Como operar o dashboard

1. Acessar o painel
2. Ver abas principais
3. Identificar:
   - erros em volume
   - erros repetidos
   - casos urgentes
4. Para cada erro:
   - validar logs e protocolos
   - aplicar tratativa
   - escalar se necessário

## O que o Renan destacou sobre o painel

- As abas principais de erro para cobrança são como Matrícula, Contrato, Agrupador e RPC. Essas representam o envio das cobranças em si. Outras abas como desconto, baixa, cancelamento, pix e cartão representam alterações de cobrança. fileciteturn3file0
- O suporte pode olhar as abas filtrando por **erro no retorno** e **erro no envio**. Erro no envio é mais raro, mas quando acontece precisa ser tratado. A primeira aba oferece um overview de todos os casos. fileciteturn3file0

## Prioridade do suporte

1. Erros recorrentes
2. Erros que impactam muitos usuários
3. Erros sem tratativa definida
4. Casos que o dashboard mostra em volume crescente

## Quando virar problema, e não chamado isolado

Se o erro:
- acontece várias vezes
- impacta múltiplos usuários
- não tem causa clara
- já gerou vários chamados parecidos

👉 Não tratar isoladamente  
→ registrar em [Problemas recorrentes](problemas.md)  
→ buscar causa raiz
